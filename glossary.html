Glossary

Every term below is deep-linkable (each has its own #term-… anchor you can bookmark or share) and every entry is fully visible in this page's plain HTML — nothing here requires JavaScript to read. Terms tagged pdf come from the source monograph's own glossary (Appendix C, pp. 66–67); terms tagged research are added from supplemental research to cover mechanisms and systems the source predates or only mentions briefly. This page works identically with JavaScript disabled; when JavaScript is available, the filter box below narrows the list live.

Without JavaScript: browse alphabetically using the letter index above, or use your browser's own find-in-page (Ctrl/Cmd+F) — every term and definition is plain, always-visible text.

A

AdamW # pdf
An Adam optimizer variant that decouples weight decay from the gradient-based moment estimates, giving more reliable regularization than plain Adam at the scale of modern LLM training. See: Ch. 13
ALiBi # pdf
A positional scheme that adds a fixed, distance-proportional bias directly to attention scores instead of encoding position in the embeddings; must be trained in from the start, but extrapolates well to sequences longer than training. See: Ch. 13
Attention # pdf
A content-addressed operation that uses a query to weight value vectors by their match with keys, letting a model select relevant information rather than compressing every input into one fixed representation. See: Ch. 8, Ch. 9
Auxiliary-loss-free load balancing # research
An MoE load-balancing approach that adjusts a per-expert routing bias directly from recent load, rather than adding an extra weighted loss term, removing a hyperparameter from the training objective. See: Ch. 14
AWQ (Activation-aware Weight Quantization) # research
A post-training quantization method that reduces model storage and memory bandwidth by identifying and protecting the weights most sensitive to quantization error. See: Ch. 15

B

Best-of-N # pdf
A test-time-compute strategy: sample N candidate outputs and use a reward or verifier model to select the single best one. See: Ch. 19

C

Causal mask # pdf
An additive mask on attention scores that forbids position t from attending to any position after t, which is what makes a Transformer a valid autoregressive language model. See: Ch. 9
Chinchilla scaling # pdf
The finding that many large language models were under-trained relative to their parameter count for their compute budget, shifting emphasis toward more training tokens per parameter. The often-quoted "about 20 tokens per parameter" is a fitted rule of thumb specific to one study's setup, not a universal constant. See: Ch. 12, Appendix D.3
CLIP # research
A vision-language model trained with a contrastive objective to align image and text representations, widely reused as an off-the-shelf visual encoder in multimodal LLM systems. See: Ch. 18
Context length # pdf
The maximum number of tokens a model can process as input in one forward pass — a serving/architecture capability, not by itself a guarantee of reliable recall across that entire length. See: Ch. 15
Cross-attention fusion # research
A multimodal fusion pattern that injects encoder features into an LLM through dedicated attention layers rather than concatenating them into the input sequence, controlling sequence-length growth (e.g., Flamingo). See: Ch. 18
Cross-entropy # pdf
The standard next-token training loss; minimizing it is mathematically equivalent to minimizing the KL divergence between the model's predicted distribution and the true conditional distribution. See: Appendix A

D

DPO (Direct Preference Optimization) # research
A post-training method that optimizes a model directly on pairwise preference data without training a separate reward model — an alternative to full RLHF. A training-procedure change, not an architecture change. See: Ch. 19

E

Embedding # pdf
A learned, dense vector representation of a discrete token (or other input unit) that places semantically or distributionally similar items nearer each other in vector space. See: Ch. 4
Expert-choice routing # pdf
An MoE routing scheme in which each expert selects the tokens it processes up to a fixed capacity, bounding per-expert load by construction — in contrast to token-choice routing. See: Ch. 14
Expert parallelism # pdf
Sharding an MoE model's experts across multiple accelerators, requiring an all-to-all communication step to move each token to its assigned expert's device and back. See: Ch. 14

F

FlashAttention # pdf
An exact (non-approximate) attention algorithm that computes scores tile-by-tile using an online/running softmax, avoiding materializing the full score matrix in slow GPU memory. See: Ch. 15, Code E.4

G

GPTQ # research
A post-training quantization method for reducing LLM weight precision (and memory footprint) while limiting accuracy loss, using layer-wise error compensation. See: Ch. 15
GQA (Grouped-Query Attention) # pdf
An attention variant in which groups of query heads share a single key/value head, trading some per-head expressivity for a smaller KV cache than full multi-head attention. See: Ch. 13
GRPO # research
A reinforcement-learning post-training method (used in DeepSeekMath/DeepSeek-R1-style training) that optimizes relative to a group of sampled outputs rather than a learned value function; a training-procedure innovation, not an architecture change. See: Ch. 19

H

Head dimension # pdf
The width of each individual attention head's query/key/value vectors, typically the model dimension divided by the number of heads. See: Ch. 9
HiPPO # pdf
A theoretical framework deriving structured state-space transition matrices that let a fixed-size recurrent state optimally compress a continuous input history, underlying S4's initialization. See: Ch. 16

I

Infini-attention # research
A long-context technique that compresses older attention information into a fixed-size compressive memory rather than discarding it or keeping it in a fully-growing cache. See: Ch. 15

K

KV cache # pdf
The stored key and value vectors for every previously generated token, reused at each new decoding step so attention need not be recomputed from scratch; grows linearly with sequence length and dominates long-context serving memory. See: Ch. 15, Appendix D.2

L

Lost in the middle # research
The empirical finding that models can attend less reliably to information placed in the middle of a long context than information near the start or end — directly relevant to where retrieved passages are placed in a RAG pipeline. See: Ch. 17

M

Mamba # research
A selective state-space model whose recurrence parameters are input-dependent per token, restoring training parallelism via an associative parallel scan rather than S4's fixed convolutional form. See: Ch. 16, Code E.6
MLA (Multi-Head Latent Attention) # research
An attention variant (introduced in DeepSeek-V3) that compresses cached key/value information for all heads into a single low-rank latent vector per token — a further point on the KV-cache-compression spectrum beyond GQA/MQA. See: Ch. 13
Model dimension (d_model) # pdf
The width of the residual stream / hidden representation carried through every layer of a Transformer. See: Ch. 9
MoE (Mixture-of-Experts) # pdf
An architecture in which a router activates only a small subset ("top-k") of many expert feed-forward sub-networks per token, decoupling total model capacity from per-token active compute. See: Ch. 14
MQA (Multi-Query Attention) # pdf
An attention variant in which all query heads share a single key/value head, minimizing KV-cache size at the cost of reduced per-head expressivity compared to GQA or full MHA. See: Ch. 13
Multi-token prediction # research
An auxiliary training objective/head that predicts several future tokens at once rather than only the next one, used alongside MoE routing in some recent systems as a training-signal enhancement. See: Ch. 14
μP (Maximal Update Parametrization) # pdf
A principled rescaling of initialization, learning rate, and layer outputs by model width that lets hyperparameters tuned on a small proxy model transfer reliably to a much larger model. See: Ch. 13

N

NTK-aware scaling # research
A RoPE context-extension technique that scales rotation frequencies unevenly — preserving short-range precision while compressing long-range behavior — rather than applying one uniform interpolation factor. See: Ch. 15

O

Online softmax # pdf
A running-maximum, running-denominator technique for computing an exact softmax incrementally over small tiles of a score row, without ever holding the full row in memory at once — the algorithmic core of FlashAttention. See: Ch. 15, Code E.4
ORM (Outcome Reward Model) # pdf
A reward model trained to judge only whether a final answer is correct, given the full reasoning trace that produced it — cheaper to supervise than a PRM but gives no per-step signal. See: Ch. 19

P

PagedAttention # pdf
A KV-cache management technique that stores cached keys/values in fixed-size, non-contiguous memory blocks (like OS virtual memory paging), reducing fragmentation and increasing serving batch size. See: Ch. 15
Perplexity # pdf
exp(cross-entropy); rescales the training loss into "effective number of equally likely next-token choices" units that are easier to reason about intuitively. See: Ch. 1
Position interpolation # pdf
A RoPE context-extension method that rescales position indices by the ratio of trained to target context length before computing rotation angles, keeping angles within the model's trained range. See: Ch. 15, Appendix D.6
Post-norm # pdf
The original Transformer normalization placement, normalizing the output of each sub-layer (including the residual sum) rather than its input; generally needs more careful warmup to train stably at large depth than pre-norm. See: Ch. 13
Pre-norm # pdf
A normalization placement that normalizes only a sub-layer's input, leaving the accumulating residual stream itself unnormalized; more stable for training very deep Transformers. See: Ch. 13
PRM (Process Reward Model) # pdf
A reward model trained to score the correctness of each individual step in a reasoning trace, giving more granular training and search signal than an outcome-only reward model, at the cost of needing step-level supervision. See: Ch. 19
Projector / adapter fusion # research
A multimodal connector strategy in which a small trainable network maps a largely-frozen encoder's features into an LLM's fusion point, letting most of the encoder and LLM stay frozen during the cheapest training stage. See: Ch. 18

R

RAG (Retrieval-Augmented Generation) # pdf
An architecture pattern that conditions generation on passages retrieved from an external, updatable index, moving mutable or long-tail facts out of model weights (parametric memory) and into a searchable store (non-parametric memory). See: Ch. 17
Residual stream # pdf
The running total that each Transformer sub-layer adds a small correction to (x ← x + f(x)); the primary reason gradients can flow across many layers without vanishing. See: Ch. 9, Ch. 13
RLHF (Reinforcement Learning from Human Feedback) # research
A post-training method that fine-tunes a model using a reward signal derived from human preference data — a training-procedure innovation applied to a fixed architecture, not a change to the forward pass. See: Ch. 19
Ring Attention # research
A distributed attention technique that shards a very long sequence across multiple devices arranged in a ring, overlapping communication with computation to scale context length beyond a single device's memory. See: Ch. 15
RMSNorm # pdf
A normalization function that scales activations by their root-mean-square magnitude only, omitting LayerNorm's mean-centering and bias term for a cheaper per-layer computation. See: Ch. 13, Code E.3
RoPE (Rotary Position Embedding) # pdf
A positional scheme that rotates query/key vectors by an angle proportional to position, so attention scores depend on relative offset even though no relative term is computed explicitly. See: Ch. 13, Code E.2
RWKV # research
A recurrent architecture designed to train with Transformer-like parallelism while running as a fast, fixed-state recurrence at inference — an alternative to both plain Transformers and SSMs. See: Ch. 16

S

Self-consistency # pdf
A test-time-compute strategy: sample multiple reasoning chains for the same problem and take a majority vote across their final answers. See: Ch. 19
Selective scan # pdf
An associative-scan algorithm that computes a selective (input-dependent) state-space recurrence's outputs for an entire sequence with logarithmic-depth parallel structure rather than one strictly serial step at a time. See: Ch. 16, Code E.6
Sliding-window attention # research
A sparse-attention pattern that restricts each token to attend only within a fixed-size local window, stacking windows across layers so information can still propagate over longer effective ranges (e.g., Mistral 7B). See: Ch. 15, Code E.7
Speculative decoding # research
A decoding-system technique that uses a small draft model to propose several tokens which a larger model verifies in parallel, speeding up generation without changing the larger model's output distribution. See: Ch. 15
SSM (State-Space Model) # pdf
A sequence-modeling family that maintains a fixed-size recurrent state updated by a (possibly input-selective) linear recurrence, trading the Transformer's unbounded KV cache for bounded memory. See: Ch. 16
SwiGLU # pdf
A gated feed-forward design that multiplies a SiLU-activated gate branch elementwise into a value branch before the output projection, outperforming a plain ReLU MLP at matched compute. See: Ch. 13, Code E.3
Switch Transformer / top-1 routing # research
The simplest MoE routing design, activating only the single highest-scoring expert per token; efficient but most sensitive to routing collapse without careful load balancing. See: Ch. 14

T

Test-time compute # pdf
Additional inference-time computation—such as sampling, search, verification, or refinement—used around a base model forward pass to improve a selected answer. See: Ch. 19
Token # pdf
The atomic discrete unit (word, subword, or character piece) a language model's vocabulary is built from and that it predicts one at a time. See: Ch. 1
Token-choice routing # pdf
The default MoE routing scheme in which each token independently selects its own top-k experts, in contrast to expert-choice routing. See: Ch. 14

V

Vision Transformer (ViT) # research
An architecture that splits an image into fixed-size patches, linearly projects each into a token-sized vector, and processes the resulting sequence with an ordinary Transformer encoder. See: Ch. 18
Vocabulary # pdf
The fixed set of discrete tokens a language model can emit or condition on; its size directly determines the cost of the final softmax layer. See: Ch. 1

W

Warmup # pdf
A short initial phase of training in which the learning rate ramps up linearly from zero, avoiding destabilizing a randomly initialized network with a large learning rate from step one. See: Ch. 13, Code E.8

Y

YaRN # pdf
A RoPE context-extension method that, like NTK-aware scaling, adjusts rotation frequencies unevenly across dimensions rather than uniformly, extrapolating to longer contexts with less quality loss than naive position interpolation. See: Ch. 15
